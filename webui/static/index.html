🆔 Communication CAN (TX + RX)

🚌 CAN0 — Cluster bus

🔍 CAN1 — OBD2 bus (UDS query)

📤 TX — UDS request to engine ECU

Frame sent every 200 ms: 7E0#03.22.39.C0.00.00.00.00

📥 RX — Expected UDS response

Expected response format: 7E8#LEN.62.39.C0.MAP_HI.MAP_LO.…

0x62 = SID 0x22 + 0x40, then DID echo, then 16-bit MAP big-endian (mbar)

📡 Architecture (cliquer pour voir)

BOT32 est le client. Le module MITM (sur un 2ème ESP32 ou hardware OpenHaldex-C6 officiel) fait le travail réel d'interception du bus Haldex entre PCM et contrôleur AWD. BOT32 ne fait que :

  • 📥 Écouter le state broadcast sur le CAN ID configuré (défaut 0x6B0)
  • 📤 Envoyer une command frame "set mode" (mode N° en byte 0 du frame configuré, défaut 0x6B1)

📖 Setup du module MITM : voir docs/haldex_integration.md

🙏 Protocole + concept : ce module dialogue avec un device externe utilisant le protocole CAN publié par OpenHaldex-C6 par Forbes Automotive. BOT32 implémente seulement le côté client (code frais MIT) et n'inclut aucun code source du projet OpenHaldex.

📶 WiFi remote (accès cellulaire dans l'auto)

Active un point d'accès WiFi pour consulter les données via téléphone

Quand activé, l'ESP32 crée un point d'accès WiFi. Connecte ton téléphone à ce réseau puis ouvre http://192.168.4.1 dans le navigateur — tu verras une page mobile avec les données live + le bouton Clear Engine Fault.
⚠ La connexion USB-C avec le PC reste 100% active en parallèle, peu importe l'état du WiFi.

📡 Détails techniques (cliquer)
  • Mode WiFi : WIFI_AP (ou WIFI_AP_STA si Haldex ESP-NOW actif aussi)
  • IP fixe : 192.168.4.1 (défaut ESP32 softAP)
  • Server HTTP sur port 80 (WebServer.h built-in, pas de lib externe)
  • Routes :
    • GET / → HTML mobile (embedded PROGMEM)
    • GET /api/status → JSON live (lever, MAP, coolant, ethanol, haldex)
    • POST /api/cmd → commandes (clear_engine_fault, set_haldex_mode)
  • Polling téléphone : 500 ms (auto-refresh JavaScript)
  • Pas d'auth — réseau privé fermé (à toi de protéger l'accès au véhicule)

🔧 Clear Engine Fault

OBD-II Mode 04 broadcast — efface les DTC d'émission

Envoie une trame OBD-II Mode 04 (broadcast 0x700 → [01 04]) qui demande à tous les ECU compatibles OBD-II d'effacer leurs DTC d'émission.
